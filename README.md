# Тестовый проект под изучение CI / CD

## Технологии, которые использовались в этом проекте:

1. **GitHub Action Pipeline**, для тестирования, сборки и отправки готового образа в GHCR.
2. Создание образа чере **Dockerfile**.
3. **Terraform**, который был реализован в Yandex.Cloud
4. **Kubernetes** кластер, который был развернут в Yandex.Cloud с 1 мастер-нодой и 1 воркер-нодой.
5. **Python-код на Flask-фреймворке**, в котором был создан один корневой маршрут (/).

---

### Python:

```
- Задача: Создать простое приложение с зависимостями на Flask и созданием отдельного файла requirements.txt.
```

- Создано: Файл зависимостей requirements.txt.
- Создано: Файл main.py содержащий основной код.
- Создано: Файл Dockerfile для сборки кода.

### Terraform:

```
- Задача: Создать удаленно с помощью кода кластер Kubernetes и работчую ноду с одним узлом для будущих подов тестового приложения.
```

Необходимо установить значения переменных окружения из идентификатора каталога, облака и генерация токена аутентификаций для сервисного аккаунта (созданного ранее).

```
export YC_TOKEN=$(yc iam create-token --impersonate-service-account-id <идентификатор_сервисного_аккаунта>)
export YC_CLOUD_ID=$(yc config get cloud-id)
export YC_FOLDER_ID=$(yc config get folder-id)
```

- Создал: Зону доступности в файле provider.tf лучше указывать ru-central1-e (на данный момент это новая зона доступности).
- Создал: Файл output.tf, который выводит резульат после сборки в консоль.
- Создал: Файл с переменными по умолчанию variables.tf
- Создал: Файл с переменными переопределяющие переменные по умолчанию terraform.tfvars

## Kubernetes:

## GitHub Actions:

```
- Задача: Создать в GitHub Actions CI/CD пайплайн для установки репозитория на воркер ноду, вход в репозитории GHCR, сборку образа, публикацию его в Registry (CI). Далее, реализуя часть CD, необходимо установить репозитории повторно в рамках новой задачи, установить файл kubeconfig в секреты GitGub Secrets, и использовать его для установки соединения с нашим кластером и установки нового образа.
```

- Создан: Файл ci.yaml реализающий данный функционал (без тестов).
